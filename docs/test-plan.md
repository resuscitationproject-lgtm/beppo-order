# テスト計画

## 顧客画面

- 受付中、準備中、休業、売り切れ、残り3/1/0食の表示とボタン制御
- API読込失敗、再試行、スケルトン、空データ
- 3つの基本メニュー、数量1〜3、残食数超過防止
- 辛さ、ビーガン不可理由、売切れトッピングの表示
- デザート・ドリンクのセット価格と合計
- 満枠表示と選択禁止
- 氏名、電話番号（ハイフン有無）、同意、備考、アレルギー検証
- 確認画面、二重クリック、タイムアウト、送信結果確認
- 完了画面の注文番号が `BYYMMDD-001` 形式
- QR流入元の許可値、未指定 `direct`、不正値 `other`
- iPhoneホーム画面、Androidインストール、オフライン案内

## 管理画面

- iPad横向き2カラム、縦向き1カラム、分割表示、横スクロールなし
- デモPIN、本番PIN、失敗、セッション期限、ログアウト
- 注文一覧の受取時間順、詳細、各状態遷移、キャンセル
- 店頭販売 −1、5秒Undo、残り0での拒否、同時操作
- 提供予定数、取り置き、本日の内容、営業状態更新
- 通常15秒、残り3食以下10秒、非表示60秒、復帰即時更新
- オフライン操作禁止、オンライン復帰、同期エラー
- 新規注文バナー、カード強調、通知音ON/OFF
- Safariバックグラウンド復帰と長時間表示

## 在庫競合

1. 残り1食で1食注文：成功し0食。
2. 残り1食で2食注文：`STOCK_SHORTAGE`、注文行なし、在庫不変。
3. 同時に1食ずつ注文：片方のみ成功。
4. 店頭販売と注文を同時実行：残数が負にならない。
5. 同じ `clientRequestId` を2回送信：1行だけ登録。
6. キャンセル：モバイル注文数と受取枠を1回だけ戻す。

## セキュリティ・個人情報

- HTMLタグを氏名・備考へ送っても実行されない
- 数量、電話、受取枠、商品ID、合計を改ざんしてもGAS側で拒否または再計算
- 平文PIN、トークン、個人情報がソースやシステムログへ出ない
- 90日経過データの氏名と電話がマスキングされる
- Chat/Gmail失敗時も注文が成功する

## 公開前の端末組合せ

最新と1世代前を目安に、iPhone Safari、iPad Safari（横・縦・standalone）、Android Chrome、デスクトップChrome/Safariで確認します。
