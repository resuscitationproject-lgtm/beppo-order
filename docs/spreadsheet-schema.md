# スプレッドシート設計

`setupSpreadsheet` が次の9シートを作ります。

| シート | 主な役割 |
|---|---|
| 営業日管理 | 日別の営業状態、提供予定数、モバイル注文、店頭販売、取り置き、本日の内容 |
| 注文履歴 | 注文、顧客連絡先、金額、状態、流入元 |
| メニュー管理 | 基本メニュー、価格、販売可否 |
| トッピング管理 | 価格、販売可否、任意の在庫制限 |
| 追加商品管理 | dessert / drink、単品価格、セット価格 |
| 受取時間管理 | 日別15分枠、上限、現在件数、受付状態 |
| 店舗設定 | 公開設定と運用数値。秘密値はScript Propertiesを優先 |
| 管理セッション | ハッシュ化したトークンと有効期限 |
| システムログ | 個人情報を含まない処理ログ |

残り食数は `提供予定数 - モバイル注文数 - 店頭販売数 - 取り置き数` です。注文作成、店頭販売、キャンセルは `LockService` 内で更新します。注文番号は同日の注文履歴から `BYYMMDD-連番` で発行します。

注文履歴の末尾に `注文明細JSON` 列を持ち、1食ごとの基本メニュー、辛さ、ビーガン、トッピング、金額を保存します。既存注文は従来列から読み取るため、移行後も表示できます。

日時は `Asia/Tokyo` を初期値とします。行の手動削除・見出し変更はAPIを壊すため避けてください。
